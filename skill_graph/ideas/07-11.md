# 2026-07-11 FlowTTE 구조 분석 및 성능 개선 아이디어

## 1. 결론

현재 성능을 크게 올릴 가능성이 가장 높은 방향은 672 grid 내부의 Flow를
더 복잡하게 만드는 것이 아니다. 유지할 중심 가설은 다음과 같다.

```text
기존 672 H+ DVT Flow coarse anchor 유지
        +
1120/1344 high-resolution shallow features
        +
support-only soft normal-subspace atlas
        +
image-disjoint normal reliability estimation
        +
cross-scale persistent component reconstruction
        ->
reliability-gated coarse/fine fusion
```

핵심은 `고해상도 관측`, `정상 변동의 mode별 모델링`, `region topology`를
서로 다른 병목으로 취급하고 단계별로 검증하는 것이다.

## 2. 현재 기준선

- Method: DINOv3-H+/16 layers `[7,15,23,31]`, `layer_norm_mean`,
  DVT-lite `position_mean alpha=1.0`, patch-wise MLP Flow, fixed 16-shot
  latent memory, no TTE, latent 1-NN + weak NLL penalty.
- Historical H+ anchor: `seg_AUROC_0.05=0.836739`, `seg_F1=0.527427`.
- Phase-3 best (`lambda_logdet=0.02`, brightness `0.8–1.2`):
  `0.837426 / 0.530635`.
- Hyperparameter tuning gain is only about `+0.000687 AUROC / +0.003208 F1`.
- Reported SuperADD `0.839300 / 0.626113` is unmatched context only. P16/Pfull,
  evaluation population, threshold, morphology, and output grid가 달라 직접적인
  superiority delta로 사용하지 않는다.

## 3. 구조적 병목

### 3.1 Score-field fragmentation와 불완전한 영역 회수

현재 contextual F1 gap은 모든 object에 균등하지 않다. `fabric`, `vial`,
`wallplugs`가 reported SuperADD 대비 per-object F1 gap 합의 약 75%를 차지한다.

| Object | Phase-3 F1 | Reported SuperADD F1 | Contextual gap |
|---|---:|---:|---:|
| fabric | 0.697950 | 0.937400 | 0.239450 |
| vial | 0.439581 | 0.647700 | 0.208119 |
| wallplugs | 0.665702 | 0.791600 | 0.125898 |

공유 morphology는 macro F1을 `0.527427 -> 0.542316`으로만 올렸지만,
`fabric`은 `0.697427 -> 0.800681`로 크게 개선했다. 따라서 topology는 실제
병목이지만 하나의 fixed morphology로는 object별 defect geometry를 처리할 수
없다.

내부 `seg_F1`은 PR curve 전체의 oracle max-F1이다. 그러므로 **객체-전역
monotonic** scalar transform은 내부 F1 상한을 높일 수 없다. 단, 이 한계는
전역 변환에만 적용된다. `seg_F1`은 객체의 모든 테스트 이미지 픽셀을 풀링한 뒤
단일 임계값으로 계산되므로 (`src/post_eval.py:335-346`), **이미지별로 다른**
변환(per-image standardization)은 이미지 간 픽셀 ordering을 바꾸어 풀드
oracle F1을 실제로 올릴 수 있다. 상세는 12절 참조. 성능을 올리려면 cross-image
score ordering, spatial evidence, 또는 component topology가 변해야 한다.

### 3.2 672/patch-16의 sub-token information loss

`can`의 median defect bbox는 약 `15x13` native pixels이다. 672/patch-16에서는
결함이 하나의 token보다 작거나 비슷해 Flow 입력 전부터 signal이 희석된다.

```text
resolution:                672   896   1120  1344  2048
grid-compatible oracle F1: .355  .465  .511  .593  .707
```

이는 resolution을 가장 강한 미소진 upstream lever로 지지한다. 단, 이 값은
decoder/scorer의 실제 성능이 아니라 observability upper bound이다. 실제 native
raw global G0의 `can` F1은 `0.017797`에 불과했고, hard-local DARC는 대량의
false positive를 만들었다. 따라서 필요한 것은 단순 고해상도가 아니라
`고해상도 + 강한 normal model + reliability control`이다.

### 3.3 Global calibration과 normal-mode 혼합

현재 scorer는 모든 support patch의 leave-one-out 1-NN distance를 하나의
global mean/std로 정규화한 뒤 NF NLL penalty를 더한다.

```text
global support LOO mean/std
-> query latent 1-NN z-score
-> + density_weight * NLL penalty
```

이 구조는 정상 변동이 큰 금속, 반사, 직물, object part를 하나의 거리 척도로
비교한다. 또한 Flow는 NLL로 학습되지만 inference의 주 score는 latent 1-NN이라
training objective와 scoring statistic이 일치하지 않는다.

과거 position-wise centering/z-score는 크게 실패했으므로 다음 방향은 absolute
position calibration이 아니다. 검색된 appearance/semantic normal mode별
conditional calibration이 필요하다.

### 3.4 Query-side normal shift

DVT 분석에서 query-side correction이 final separation 회복에 대부분 기여했다.
따라서 query nuisance mismatch는 실제 병목이다. 그러나 다음 naive adaptation은
이미 실패했다.

- ungated TTE memory expansion
- rotation-8 support augmentation
- image-level top-M sub-bank routing
- support-consensus residual

향후 adaptation은 query를 memory에 삽입하지 않고, low-score normal consensus로
추정한 제한된 nuisance transform만 이미지 단위로 적용해야 한다.

## 4. 최우선 방법: High-Resolution Soft Subspace Atlas

### 4.1 Coarse branch

현재 672 H+ DVT Flow score `s_coarse`를 그대로 유지한다. 이 branch는 강한
continuous ranker이므로 교체 대상이 아니라 no-harm anchor이다.

### 4.2 Fine normal atlas

1120 또는 1344에서 shallow/mid-level H+ features를 추출한다. Support token을
XY coordinate가 아니라 appearance와 detached coarse semantic context로 soft
grouping한다.

각 normal mode `k`에 대해 다음을 적합한다.

- centroid `mu_k`
- shrinkage covariance
- PCA normal subspace `U_k`
- image-disjoint held-out normal residual dispersion `sigma_k`

Mode별 score:

```text
d_k(f) = ||(I - U_k U_k^T)(f - mu_k)|| / (sigma_k + eps)
```

Hard minimum은 anomaly를 다른 normal mode로 설명할 위험이 있으므로 posterior
weighted log-sum-exp 또는 제한된 soft-min으로 marginalize한다.

```text
fine token
-> appearance/coarse-context mode posterior
-> mode별 whitened orthogonal residual
-> soft marginalization
-> s_fine
```

### 4.3 기대 효과

- `can`: high resolution으로 sub-token cue 복원.
- `fabric`: 반복 직물의 정상 variation direction을 subspace 안으로 흡수.
- `vial`: 조명과 반사 variation을 normal manifold direction으로 제거.
- `wallplugs`: 서로 다른 part appearance를 복수 normal mode로 분리.
- 학습과 추론을 동일한 reconstruction residual 통계로 정렬.

단순 global PCA는 SubspaceAD와 중복되므로 novelty가 아니다. 후보 novelty는
`multi-resolution soft atlas + coarse-semantic conditioning + normal-only
uncertainty gate + coarse no-harm fusion`의 결합에 있다.

## 5. Reliability gate와 fusion

Fine score를 그대로 max-fusion하면 high-resolution normal noise까지 증가할 수
있다. Reliability는 AD2 label이 아니라 image-disjoint normal folds로만 추정한다.

후보 입력:

- mode posterior entropy
- held-out normal residual dispersion
- nearest-normal support-image disagreement
- cross-resolution rank consistency
- tile/view agreement

```text
reliability r(x) in [0,1]
s_fused(x) = max(s_coarse(x), r(x) * s_fine(x))
```

Gate는 반드시 다음 hard null을 이겨야 한다.

- `fine only`
- ungated `max(coarse, fine)`
- normal-calibrated scalar gate
- shuffled gate
- inverse gate

제안 gate가 ungated max와 scalar gate를 이기지 못하면 복잡한 spatial gate의
novelty를 폐기한다.

## 6. Cross-Scale Persistent Components

Threshold 후 fixed morphology 대신 여러 scale과 view에서 반복되는 component를
이용해 continuous score를 재순위화한다.

```text
672 / 896 / 1120 / 1344
x identity / weak photometric views
x overlapping tile phases
-> native coordinates로 정렬
-> component persistence와 agreement 계산
-> component-level normal tail score
-> reconstructed continuous map
```

Component features:

- resolution 간 persistence
- tile-boundary 변경에 대한 안정성
- photometric-view 안정성
- coarse/fine 동시 반응
- peak score와 normal-tail probability
- boundary agreement

이는 monotonic calibration이 아니라 pixel/component ordering을 변경하므로
oracle F1도 개선할 수 있다. 다만 spatial dependence 때문에 pixel-level
conformal guarantee를 component-level guarantee로 과장하지 않는다.

## 7. 후속 후보

### 7.1 Coarse-Conditioned Predictive Residual

Deep coarse feature `c`로 high-resolution shallow feature `f`의 정상값과 분산을
예측한다.

```text
(f_hat, sigma_hat) = g(stopgrad(c))
s_pred = ||f - f_hat|| / (sigma_hat + eps)
```

정상 illumination/material shift가 coarse와 fine에 함께 나타나면 예측되지만,
coarse token에서 희석된 작은 scratch가 shallow feature에만 나타나면 큰 residual이
생긴다는 가설이다. Soft atlas가 positive일 때만 더 복잡한 후속 head로 검토한다.

### 7.2 Query-Constrained Normal Transport

Query를 memory에 추가하지 않는다. Coarse/fine/augmentation 모두에서 low-score인
patch만 candidate normal로 사용해 support prototype의 제한된 low-rank nuisance
transform을 추정한다.

안전장치:

- support PCA hull 밖 이동 금지
- high-score patch stop-gradient
- image마다 adaptation reset
- static branch 유지
- pre/post transport anomaly response retention 측정

FastRef-style query refinement와 중복될 수 있으므로 `cross-resolution consensus`,
`support-subspace trust region`, `static no-harm anchor`가 없으면 신규성 주장을 하지
않는다.

### 7.3 Pseudo-Defect Boundary Decoder

Label-free atlas로도 F1이 충분히 오르지 않을 때만 고려한다. Frozen coarse score,
high-resolution residual, RGB edge, scale consistency, uncertainty를 입력으로 하는
작은 segmentation decoder를 normal-only synthetic thin defects로 학습한다.

가장 큰 F1 상승 가능성이 있지만 synthetic shortcut, P16 overfitting, GLASS/RealNet
등과의 novelty 중복 위험이 크므로 마지막 선택지로 둔다.

## 8. 재시도하지 않을 방향

저장소 증거상 다음은 현재 main branch에서 중단한다.

- 672 grid 내부의 더 복잡한 Flow, Transformer, Conv2D.
- CLS/register/coordinate conditioning과 image top-M routing.
- support-consensus 또는 quantile residual.
- hard foreground/background suppression.
- support-position centering/z-score와 local contrast.
- threshold 또는 morphology만을 중심으로 한 개선.
- hard local window, RANSAC registration, robust reconstruction.
- ungated test-time memory expansion.

## 9. 단계별 판정 실험

### Gate 1 — Scorer hard null

동일 token, support, compute budget에서 비교한다.

```text
kNN mean/quantile
global PCA residual
shrinkage Mahalanobis
soft PCA atlas K={2,4,8}
small Flow density
```

KEEP:

- best simple null 대비 AP absolute `+0.01` 이상.
- relative AP `+20%` 이상.
- paired 95% bootstrap lower bound `>0`.
- held-out normal FPR@normal-p99.9 악화 없음.
- TPR@FPR `1e-4` 개선.

Global PCA가 현재 kNN을 이기지 못하면 atlas 확장을 중단한다.

### Gate 2 — Resolution causality

672/896/1120/1344에서 scorer, support, tile aggregation을 동일하게 유지한다.

KEEP:

- 최소 두 개 연속 resolution에서 AP/component recall 증가.
- best-vs-672 AP `+0.01` 이상.
- component recall `+0.10` 이상.
- paired lower bound `>0`.
- broad-control pAUROC loss `<=0.005`.

### Gate 3 — Reliability identification

Proposed spatial gate가 ungated max와 scalar normal calibration을 모두 이겨야 한다.
Normal FPR@`1e-4`를 낮추면서 anomaly recall의 95% 이상을 보존하지 못하면 gate를
폐기한다.

### Gate 4 — AD2 shadow no-harm

- P16-random seeds `0,1,2`.
- mean oracle F1 `+0.03` 이상.
- `fabric/vial/wallplugs` 중 두 object 이상 각 F1 `+0.03`.
- `can` AP `+0.005` 또는 component recall `+0.10`.
- 어떤 object도 AP/F1 `-0.02` 초과 손실 금지.
- mean pAUROC loss `<=0.002`.
- normal-calibrated fixed F1도 개선. Oracle-only gain이면 deployment claim 중단.

### Gate 5 — Untouched claim

P16-random 3 seeds와 Pfull을 분리하고 동일 evaluator/population/postprocess 아래에서
비교한다.

- retained Flow anchor
- global PCA/Subspace hard null
- matched SuperAD/SuperADD-family baseline
- proposed atlas/fusion

Paired bootstrap lower bound, fixed-threshold performance, runtime, memory를 함께
보고한다. Public AD2 결과를 보고 gate나 resolution을 재조정하면 final claim은
무효로 처리한다.

## 11. 최종 선택

단일 main hypothesis는 다음으로 고정한다.

> 기존 H+DVT coarse ranker를 보존하고, 고해상도 token에 soft normal-subspace
> atlas를 구성한 뒤, image-disjoint normal dispersion으로 fine evidence를 gating한다.

이는 현재 preregistration의 모호한 `learned high-resolution density head`를
train/inference-aligned normal geometry로 구체화한다. 성능과 논문 기여를 모두
정당화하려면 learned head가 raw cosine뿐 아니라 global PCA, shrinkage
Mahalanobis, kNN quantile을 동일 protocol에서 이겨야 한다.

### 12.2 평가 메커니즘에서 도출된 두 가지 미공략 결함

코드 확인 결과:

- **결함 A — 객체-전역 풀드 임계값 vs 이미지별 스케일 드리프트.**
  `seg_F1`은 객체의 모든 테스트 이미지 픽셀을 하나로 풀링한 뒤 단일 oracle
  임계값으로 계산된다 (`src/post_eval.py:335-346`). 이미지별 정상-배경 스코어
  레벨이 다르면 어떤 단일 임계값도 전 이미지를 만족하지 못한다. 현재
  파이프라인에는 query-이미지별 스코어 정규화가 전혀 없다. 지금까지 killed된
  보정 계열(support-position center/zscore)은 전부 위치별 보정이었고,
  이미지별 전역 보정은 시험된 적이 없다.
- **결함 B — patch-16 bilinear 업샘플의 경계 램프.** 스코어맵은 토큰 그리드
  (672 기준 42x42)에서 `cv2.INTER_LINEAR`로 원본 해상도로 업샘플된다
  (`scripts/flow_tte_mvtec_ad2_core.py:508`). 결함 경계마다 약 16px 소프트
  램프가 생기고, 둘레/면적 비가 큰 클래스(fabric, sheet_metal)일수록 임계값
  주변 픽셀 손실이 커진다. SuperADD는 H/4 x W/4 그리드에서 마스크를 만든다.

### 12.3 제안 0 — 갭 분해 진단 (최우선, 반나절, GPU 최소)

앵커 맵을 1회 재생성한 뒤 동일 맵에 대해 세 가지 F1을 계산한다.

1. 전역 oracle F1 (현행): `0.5274`
2. 이미지별 oracle F1 (이미지마다 최적 임계값 허용):
   전역과의 차이 = **스케일 드리프트 기여분** (결함 A의 이득 상한)
3. 경계-관용 F1 (GT ±8px 관용 대역):
   차이 = **경계 램프 기여분** (결함 B의 이득 상한)

이 분해로 격차 `0.0987`이 `드리프트/경계/진짜 미검출`로 정량화되고 12.4의
각 아이디어 기대 수익이 사전 계산된다. 만약 이미지별 oracle F1이 전역과
비슷하면 아이디어 1은 즉시 폐기한다 (반증 가능). AD2 public은 shadow이므로
진단 용도로 허용된다.

### 12.4 스코어-분포 등가화 계열 아이디어 (killed 방향과 비중복)

- **아이디어 1 — query-이미지별 robust 스코어 표준화 (결함 A 직격).**
  각 query 이미지의 하위 분위(하위 60-80% 픽셀) median/MAD로
  `(s - mu_img) / sigma_img` 표준화, 또는 support 스코어 분포로
  quantile-matching. 라벨-프리, 클래스-불가지, 위치-비의존이므로 killed된
  위치별 감산 보정과 메커니즘이 다르다. 리스크는 대형 결함이 이미지 대부분을
  덮을 때 배경 통계 오염이며, robust 하위-분위 통계가 필수다. fabric처럼
  랭킹이 마스크보다 훨씬 좋은 클래스에서 최대 회수를 예상한다.
- **아이디어 2 — 구조-유도 스코어맵 정제 (결함 B 직격).** bilinear 업샘플을
  guided filter / joint bilateral(RGB 또는 저층 feature guidance)로 교체해
  스코어 경계를 이미지 에지에 스냅. killed된 `local_contrast`는 스코어 필드
  자체의 국소 변동을 증폭했지만 이것은 이미지 구조에 정렬된 평활화라 정반대
  메커니즘이다. 학습 불필요, 비용 무시 가능, 랭킹 보존적.
- **아이디어 3 — 이웃-국소 밀도 정규화 거리 (fragmentation 직격).** raw
  latent 1-NN 거리를 검색된 이웃의 support 국소 밀도로 정규화:
  `d(q, nn1) / mean_k d(nn1, kNN_support)`. 균질 영역과 텍스처 영역의 거리
  스케일을 등가화한다. 위치 기반이 아닌 feature-공간-국소 정규화이므로 killed
  방향이 아니며, SuperADD의 layer-wise scale normalization의 대응물이다.
  3.3절의 "conditional calibration" 방향과 동일 목표의 가장 값싼 구현이다.
- **아이디어 4 — query 그리드-시프트 앙상블.** 입력을 (0,8)x(0,8)px 시프트한
  4개 뷰에서 특징 추출 후 정렬-평균해 유효 stride 16 -> 8. killed된
  rotation-8은 support 뱅크를 8배로 불려 실패했지만 이것은 support 고정,
  query 샘플링만 조밀화라 실패 메커니즘이 다르다. 비용 4x. 경계 램프 절반화와
  sub-token 결함 다중 관측을 동시에 얻는다.

### 12.5 우선순위와 4절 main hypothesis와의 관계

| 순서 | 항목 | 비용 | 표적 | 이득 상한 결정자 |
|---|---|---|---|---|
| 0 | 갭 분해 진단 | 반나절 | 격차 정량 분해 | - |
| 1 | 이미지별 표준화 | 스코어링 수정 | fabric/wallplugs/rice | 진단 (2) |
| 2 | guided 정제 | 후처리 1단계 | fabric/sheet_metal 경계 | 진단 (3) |
| 3 | 밀도 정규화 거리 | 스코어링 수정 | vial/fruit_jelly 파편화 | 중간 |
| 4 | 그리드-시프트 | 4x 추론 | 경계 + 소결함 | 중간 |

아이디어 1+2+3은 서로 직교(이미지 간 분포 정렬 / 경계 / 영역 간 스케일
등가화)하며 전부 클래스-불가지, 정상-전용, all-8 규약을 지킨다. 이 계열은
4-6절의 high-resolution soft atlas 가설과 경쟁하지 않고 **coarse anchor
자체의 operating surface를 값싸게 회복**하는 보완 축이다. 핵심 논리 전환:

> 지금까지 표현(flow/조건화/문맥) 변경은 전부 실패했다. 12.1-12.2의 증거는
> 격차가 표현이 아니라 "스코어 분포의 이미지 간·영역 간 비등가성 + 경계
> 양자화"에 있음을 가리킨다. 진단 0이 이 가설 자체를 반증 가능하게 만든다.
