# FlowTTE 구조 분석 및 성능 개선 실험 계획안

## 1. 연구 목표

현재 FlowTTE의 남은 성능 병목을 단순한 Flow head 표현력 부족으로 가정하지 않는다. 기존 실험에서 Conv2D Flow, Transformer Coupling Flow, layer-wise NF, hard image routing이 모두 all-8 성능을 개선하지 못했기 때문이다.

본 계획의 중심 가설은 다음과 같다.

> FlowTTE의 낮은 segmentation F1은 주로 이미지 간 score-scale drift, feature-space의 정상 밀도 불균형, patch-grid 경계 양자화에서 발생한다. 이 병목을 먼저 정량적으로 분해한 뒤, 고해상도 feature가 독립적인 anomaly evidence를 제공하는 경우에만 coarse-conditioned soft subspace atlas로 확장한다.

전체 구조는 다음과 같다.

```text
672 H+ DVT FlowTTE coarse anchor
    │
    ├─ 이미지별 score-scale drift 보정
    ├─ support local-density normalized retrieval
    ├─ structure-guided boundary refinement
    └─ grid-shift/resolution observability 검증
              │
              └─ positive evidence가 있을 때만
                    1120/1344 high-resolution feature
                    + coarse-conditioned soft subspace atlas
                    + normal-only reliability fusion
                    + cross-scale persistent components
```

# 6. Phase 1: 이미지별 robust score normalization

## 가설

각 query image의 정상 score offset 및 scale이 달라 object-pooled 단일 threshold가 실패한다.

## 6.1 기본 방식

이미지 score map `s`에서 하위 분위 영역만 사용한다.

```text
B_q = {i | s_i ≤ quantile(s, q)}

μ_img = median(s[B_q])
σ_img = 1.4826 × MAD(s[B_q]) + ε

s'_i = (s_i - μ_img) / σ_img
```

Sweep:

```text
q ∈ {0.5, 0.6, 0.7, 0.8}
```

## 6.2 Support-shrinkage 방식

Query-only normalization의 과보정을 막기 위해 support calibration과 결합한다.

```text
μ = (1-λ) μ_support + λ μ_img

log σ =
(1-λ) log σ_support
+ λ log σ_img

s'_i = (s_i - μ) / σ
```

Sweep:

```text
λ ∈ {0, 0.25, 0.5, 0.75, 1.0}
```

## 6.3 안전장치

```text
σ_img ≥ support-derived MAD floor
대형 결함 이미지 별도 평가
normal image variance 확대 여부 확인
normal-image FPR 반드시 보고
```

## 6.4 비교군

```text
No normalization
Support-only global normalization
Query mean/std
Query median/MAD
Lower-quantile median/MAD
Support-query shrinkage
Support quantile matching
```

## 6.5 KEEP 조건

```text
pooled oracle F1 +0.015 이상
fixed-threshold F1도 개선
normal-image FPR 악화 없음
fabric 또는 wallplugs F1 +0.03 이상
```

Oracle F1만 개선하고 fixed threshold가 개선되지 않으면 deployment contribution으로 유지하지 않는다.

# 8. Phase 3: Support local-density normalized retrieval

## 가설

전체 support latent bank에서 1-NN distance의 의미가 normal mode별 밀도에 따라 다르다.

```text
dense normal region의 작은 거리
sparse normal region의 큰 거리
```

를 동일 global scale로 비교하는 것이 F1을 악화시킨다.

## 8.1 Support density 정의

각 support latent `n`에 대해:

```text
ρ_K(n) =
median_{j=2,...,K+1} ||n - NN_j(n)||
```

자기 자신은 제외한다.

Sweep:

```text
K ∈ {3, 5, 10, 20}
aggregation ∈ {mean, median, trimmed mean}
```

## 8.2 Query score

Query `q`의 nearest support `n*`에 대해:

```text
s_density(q) =
||q - n*|| / (ρ_K(n*) + ε)
```

안정화:

```text
ρ_K 하한: support p1
score 상한: support LOO p99.9 기반 clipping
```

## 8.3 확장 비교

```text
Raw 1-NN
Density-normalized 1-NN
kNN mean distance
kNN quantile distance
Shrinkage Mahalanobis
Global PCA residual
```

Query-side density는 anomaly cluster의 자기정상화 위험 때문에 첫 단계에서 사용하지 않는다.

## 8.4 분석 항목

```text
mode별 normal score 분산
nearest-support density와 FP의 관계
vial/fabric/specular region 성능
support image별 density bias
```

## 8.5 KEEP 조건

```text
pixel AP +0.01 이상
mean F1 +0.015 이상
TPR@FPR 1e-4 개선
normal p99.9 FPR 악화 없음
vial 또는 fruit_jelly F1 +0.03 이상
```

# 9. Phase 4: Query grid-shift ensemble

## 가설

Patch-16 sampling phase 때문에 작은 결함과 경계가 특정 grid alignment에서 희석된다.

## 9.1 2-view smoke

먼저 두 phase만 비교한다.

```text
(0,0)
(8,8)
```

각 phase의 query score map을 원래 좌표로 정렬한 뒤 결합한다.

```text
S_mean = mean(aligned score maps)
S_max  = max(aligned score maps)
S_cons = mean + γ × cross-view agreement
```

## 9.2 Phase mismatch 점검

Query phase만 바꾸면 support bank와 sampling mismatch가 생길 수 있다. 따라서 정상 support image에 같은 shift를 적용해 phase별 score drift를 측정한다.

비교:

```text
A. fixed original support bank
B. phase-calibrated score
C. phase-matched fixed support bank
```

B 또는 C가 필요하지 않다면 더 단순한 A를 유지한다.

## 9.3 4-view 확장

2-view에서 positive signal이 있을 때만:

```text
(0,0), (0,8), (8,0), (8,8)
```

으로 확장한다.

## 9.4 KEEP 조건

## 11.3 Mode routing

Anomaly feature가 자신을 가장 잘 설명하는 normal mode를 직접 선택하는 것을 막기 위해 routing feature와 residual feature를 분리한다.

```text

```

즉 high-resolution fine feature 자체로 mode를 선택하지 않는다.



## 11.4 Mode별 normal model

각 mode `k`에 대해:

```text
centroid μ_k
PCA basis U_k
eigenvalues Λ_k
orthogonal residual dispersion σ⊥,k
in-subspace dispersion σ∥,k
```

Orthogonal score:

```text
s⊥,k(f) =
||(I-U_kU_kᵀ)(f-μ_k)|| / (σ⊥,k + ε)
```

In-subspace score:

```text
a_k = U_kᵀ(f-μ_k)

s∥,k(f) =
sqrt(a_kᵀ(Λ_k + δI)^-1 a_k)
```

결합:

```text
d_k(f) = s⊥,k(f) + β s∥,k(f)
```

Sweep:

```text
β ∈ {0, 0.25, 0.5, 1.0}
```

---

## 11.5 Soft marginalization

무제한 hard minimum은 anomaly absorption 위험이 있으므로 사용하지 않는다.

```text
Top-L modes according to p(k|c)
L ∈ {1, 2}
```

Posterior-weighted score:

```text
s_fine =
-τ log Σ_{k∈Top-L}
p(k|c) exp(-d_k/τ)
```

또는 weighted mean을 hard null로 둔다.

```text
Σ p(k|c)d_k
```

안전장치:

```text
posterior entropy 기록
mode support coverage 제한
minimum posterior threshold
K 증가에 따른 normal/anomaly score 감소량 분석
```

# 12. Phase 7: Normal-only reliability fusion

Fine branch가 독립적으로 유효한 경우에만 진행한다.

## 12.1 Score calibration

Coarse와 fine score를 직접 max-fusion하지 않는다. Image-disjoint support fold를 이용해 normal-tail scale로 변환한다.

```text
u_c = -log P_normal(S_c ≥ s_c)
u_f = -log P_normal(S_f ≥ s_f)
```

Calibration은 support image-disjoint 방식으로 수행한다.

## 12.2 Reliability 입력

```text
mode posterior entropy
held-out normal residual dispersion
cross-resolution rank consistency
grid-shift/view agreement
nearest-support-image disagreement
```

첫 버전은 학습형 gate가 아니라 deterministic normal-only gate로 구성한다.

```text
r ∈ [0,1]
```

## 12.3 Fusion

추천 no-harm residual fusion:

```text
s_fused =
u_c + r × ReLU(u_f - τ_f)
```

의미:

```text
coarse anchor는 항상 유지
fine evidence가 normal tail을 충분히 초과할 때만 가산
```

비교군:

```text
Coarse only
Fine only
Ungated max
Ungated sum
Scalar normal-calibrated gate
Proposed reliability gate
Shuffled gate
Inverse gate
```

## 12.4 KEEP 조건

```text
ungated max와 scalar gate를 모두 개선
normal FPR 감소 또는 유지
anomaly recall 95% 이상 보존
mean F1 +0.02 이상
어떤 object도 -0.02 초과 하락 없음
```

복잡한 spatial gate가 scalar gate를 이기지 못하면 폐기한다.

# 14. 보류 후속 후보

## 14.1 Coarse-conditioned predictive residual

Soft atlas가 positive일 때만 비교한다.

```text
(f_hat, sigma_hat) = g(stopgrad(c))

s_pred =
||f - f_hat|| / (sigma_hat + ε)
```

Atlas 대비 추가 학습 복잡성이 정당화되는 경우에만 유지한다.

## 14.2 Query-constrained normal transport

Query를 memory에 추가하지 않는다.

```text
low-score consensus patch만 사용
support subspace trust region 내 low-rank transform
high-score patch stop-gradient
image마다 adaptation reset
static coarse branch 유지
```

이는 score calibration, density correction, atlas가 모두 부족할 때 후속으로 검토한다.

## 14.3 Pseudo-defect boundary decoder

가장 마지막 단계로 둔다.

입력:

```text
coarse score
fine residual
RGB/shallow-feature edge
scale consistency
uncertainty
```

학습:

```text
normal-only synthetic thin defects
```

GLASS/RealNet 계열과의 차별성과 synthetic shortcut 위험을 별도로 검토해야 한다.

# 17. 최종 주장 후보

모든 gate를 통과한 경우에만 다음과 같이 주장한다.

> 기존 FlowTTE의 coarse anomaly ranking을 보존하면서, 이미지 간 score drift와 mode-dependent normal density를 support-only calibration으로 보정한다. 고해상도 fine feature가 독립적인 anomaly evidence를 제공할 때는 detached coarse semantics로 normal subspace mode를 선택하고, image-disjoint normal reliability를 이용해 fine residual을 coarse score에 안전하게 결합한다.

예상 방법명:

```text
Reliability-Gated Multi-Resolution Normal Atlas
for Flow-LatentBank
```

또는:

```text
Coarse-Anchored Soft Subspace Atlas
```

단, 실제 최종 방법은 실험 결과에 따라 단순화한다.

```text
Normalization만 유효:
  score-equivalent FlowTTE

Density correction까지 유효:
  density-calibrated Flow-LatentBank

Atlas까지 유효:
  coarse-anchored soft subspace atlas

Persistence까지 유효:
  cross-scale persistent normal atlas
```

가장 복잡한 방법을 목표로 하지 않고, **가장 단순하면서 preregistered hard null을 이기는 구성**을 최종 방법으로 선택한다.
